# 板材标准分段表重构方案

> 状态:方案已确认,待实现。对应需求:`板材标准.txt`(权威标准源)。
> 关联代码:[src/dfm/standards.ts](src/dfm/standards.ts)、[src/dfm/types.ts](src/dfm/types.ts)、[src/index.ts](src/index.ts)、[iframe/material-selector.html](iframe/material-selector.html)。

## 一、背景(为什么改)

`standards.ts` 的 `JLC_MATERIAL_STANDARDS` 标准值是早期编的近似值,与权威的 `板材标准.txt` 大面积不符(板厚/线宽/过孔/焊盘/尺寸几乎都对不上)。虽然 14 处检查项都读了 `JLC_MATERIAL_STANDARDS[selectedMaterial]`(选不同板材标准确实会变),但值不对,等于没用对。

另有真 bug:[iframe/material-selector.html:133](iframe/material-selector.html#L133) 选项 `value="CEM-1"` 与 standards.ts 键 `CEM1` 不匹配 → 选 CEM-1 时取不到标准。

`板材标准.txt` 第 141 行明确:**各板材只有部分参数特有(层数/尺寸/板厚/铜厚/过孔/线宽/焊盘到线),其余共用(字符/焊环/BGA焊盘直径等)**;且部分特有参数**随层数或铜厚变化**(FR4 最大尺寸按层数 4 档、线宽按铜厚 7 档、过孔/焊盘按单/双/多层)。

**决策**:① 分段表(彻底);② 删除 CEM-1;③ 共用字段提取统一常量。

## 二、依赖确认(两维度数据源已就绪)

- **层数**:`parseCopperLayerCount()`([src/index.ts:600](src/index.ts#L600)) → `eda.pcb_Layer.getTheNumberOfCopperLayers()` ✓
- **铜厚**:`parseLayerPhysMap()`([src/index.ts:536](src/index.ts#L536)) 从文档源解析 `LAYER_PHYS` + `copperMilToOz()`([src/index.ts:524](src/index.ts#L524)) 转 oz;第 5 项外层铜厚检查已在用 ✓

两维度都能读到,分段表可行。

## 三、类型重构([src/dfm/types.ts](src/dfm/types.ts))

`MaterialStandard` 拆分:特有字段(部分改分段表)+ 共用字段移除(改用 `SHARED_DFM_STANDARDS`)。

```ts
interface SizeByLayers { minLayers: number; max: { width: number; length: number }; }
interface TraceSpecByCopper { copperOz: number; minWidth: number; minSpacing: number; }
interface ViaSpec {
  throughHole?: { min: number; max: number };
  blindHole?: { min: number; max: number };     // HDI
  buriedHole?: { min: number; max: number };    // HDI
  minPadOuter?: number;
}
interface ViaSpecByLayers { minLayers: number; spec: ViaSpec; }

interface MaterialStandard {
  name: string;
  layerCount: { min: number; max: number };
  minSize: { width: number; length: number };
  maxSize: { width: number; length: number };
  maxSizeByLayers?: SizeByLayers[];                 // FR4 按层数
  thicknessRange: { min: number; max: number };
  thicknessValues?: number[];                       // 离散板厚(高频/铝/铜)
  outerCopperThickness: number[];
  innerCopperThickness?: number[];
  viaTypes: ('through' | 'blind')[];
  viaSpec?: ViaSpec;                                // 非 FR4 单一
  viaSpecsByLayers?: ViaSpecByLayers[];             // FR4 按层数
  slotWithCopperByLayers?: Array<{ minLayers: number; width: number; length: number }>;
  slotWithCopperMinSize?: { width: number; length: number };
  slotWithoutCopperMinWidth: number;
  traceSpecsByCopper?: TraceSpecByCopper[];
  minTraceWidth?: number; minTraceSpacing?: number;
  minPadToLineSpacing: number;
  minBgaPadToLineSpacing: number;
}

/** 各板材共用参数(字符/焊环/BGA焊盘直径) */
export const SHARED_DFM_STANDARDS = {
  minStringHeight: 0.8,
  minStringWidth: 0.15,
  minStringToCopperSpacing: 0.15,
  minPluginPadRingWithCopper: 0.15,
  minPluginPadRingWithoutCopper: 0.2,
  minBgaPadDiameter: 0.25,
};
```

## 四、解析函数(新增,放 [src/dfm/standards.ts](src/dfm/standards.ts))

每个:优先按 `layerCount` / `copperOz` 查分段表,无分段表回退单值。

```ts
resolveMaxSize(s, layerCount): { width, length }        // maxSizeByLayers 取 ≥minLayers 最大档,否则 maxSize
resolveViaSpec(s, layerCount): ViaSpec                  // viaSpecsByLayers 取档,否则 viaSpec
resolveSlotWithCopper(s, layerCount): { width, length } // slotWithCopperByLayers 取档,否则 slotWithCopperMinSize
resolveMinTrace(s, copperOz): { width, spacing }        // traceSpecsByCopper 按 oz 匹配,否则单值
```

## 十、风险与简化

- **线宽按铜厚取层**:线在内/外层铜厚不同。本次**简化为按外层铜厚统一取线宽标准**(外层铜厚 `parseLayerPhysMap` 已能读;内层线按外层标准偏保守,可接受)。
- **HDI 盲/埋孔检查**:**维持原有逻辑不变**(第 8 项过孔检查按原办法处理盲埋孔),本次不新增/简化;盲/埋孔规格仅作为 `viaSpec` 数据保留供 `standardValue` 显示。
- **改动面大(类型 + 5 板材 + 4 函数 + 12 检查项)**:逐项改易引入回归,需每项实测;`sizeRange → minSize/maxSize` 等字段更名会牵连多处,重构后先确保 compile 再逐项验证数值。
